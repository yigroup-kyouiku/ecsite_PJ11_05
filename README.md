# ECサイトプロジェクト

## 環境構築
### GitHubにあるプロジェクトをクローンする（リモートにあるプロジェクトを自分のPCに複製する）
1. GitHubにログインし、"ecsite-project"のリポジトリのページを開く
2. Codeと書かれた緑色のボタンを押下し、Localのタブを開く
3. HTTPSのタブを開いて書かれているURLをコピーする
4. コマンドプロンプト（またはターミナル）を開いて以下のコマンドを実行する
```bash
git clone コピーしたURL
```

### Eclipseにプロジェクトをインポートする
1. パッケージエクスプローラーの空いている場所で右クリックし、インポートを選択
2. Mavenと書かれたフォルダマークを開き、「既存Mavenプロジェクト」を選択
3. ルートディレクトリの項目の参照をクリックし、クローンしたプロジェクトを選択し完了を押す

### pgAdminでの接続設定
1. 左のサイドバーにある「Server」を右クリックし、「登録」を選択
2. 一般タブの名前は任意でつけてください（わかりやすくECsiteなど）
3. 接続タブを選択し、以下の項目を入力してください

| 項目 | 値 |
| --- | --- |
| ホスト名 | localhost |
| ポート番号 | 5432 |
| ユーザー名 | postgres |
| パスワード | postgres |
| パスワードを保存 | ON |

4. サーバの設定完了後にサイドバーの「データベース」を右クリックし、作成→データベースを選択
5. 「ecsite」という名前で登録する

## Gitの使い方

**重要**：gitのコマンドを使用する際はコマンドプロンプト（ターミナル）でプロジェクトディレクトリに移動してから実行してください
```bash
// cdはディレクトリを移動するコマンド
cd プロジェクトディレクトリのパス（例： /Users/username/ecsite-project）
```

### 作業を始める時の操作
- 自分が所属しているチームのブランチから作業ブランチを作成してください（例：チームAの場合**teamA**ブランチから作成する）
- チームのブランチに切り替えてからブランチ作成のコマンドを打つことで自分のチームのブランチを元に作業ブランチを作成することができます

### 作業が完了しレビューしてもらうまでの流れ
- 作業が完了したら**add**、**commit**、**push**のコマンドを用いてリモートに自分の変更を反映させます
- 上記の作業を行った後GitHubのPull requestから自分のチームのブランチに向けて作業ブランチを反映させるようにプルリクエストを作成します
- プルリクエストを作成したらGoogleチャットでリーダーに報告してください

### よく使うコマンド集
```bash
// ブランチの一覧、現在いるブランチを確認できる
git branch
```
``` bash
// ブランチを切り替える
git checkout 既存のブランチ名
```
```bash
// 新しくブランチを作成する（任意のブランチ名を入力してください 例：git checkout -b ログイン機能）
git checkout -b ブランチ名
```
```bash
// 変更した内容をステージに乗せる（変更した内容をリモートに反映させる準備をする）
git add -A
```
```bash
// 変更した内容をコミットする（ステージの内容を、履歴としてローカルのGitに確定保存する）
git commit -m "コミットメッセージ"
```
```bash
// リモートに変更内容を反映させる
git push
```
